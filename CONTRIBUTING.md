# Contributing

Thanks for helping improve Telegram Music Bot.

## Local workflow

1. Fork the repository.
2. Create a feature branch.
3. Make your changes.
4. Run tests or at least verify the project builds.
5. Open a pull request.

## Guidelines

- Keep changes focused and minimal.
- Prefer configuration over code changes when possible.
- Update README when user-facing behavior changes.
- Do not commit secrets or generated tokens.

## Notes

- The project supports Docker Compose and systemd deployment.
- Standard configuration lives in `config/config.yaml`.
